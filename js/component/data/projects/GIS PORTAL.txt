											&emsp;&ensp;In order to create a logging system for on-site surveys regarding mobile and TETRA networks, an interactive/simple/intuitive interface is needed. For this, the creation of a page in CSDB is shown as the most suited solution.
											<h5><b>Objectives</b></h5>
											<ul id="first">
												<li>Data insertion and manipulation for on-site surveys data;</li>
												<li>Generalized interface complying with TETRA, GSM, UMTS and LTE network parameters;</li>
												<li>Clear view of the optional and obligatory fields to create/edit a measurement;</li>
												<li>Multiple measurements can be made by floor;</li>
												<li>Network coverage overall level based on the measurements fields.</li>
											</ul>
											<img src="img/portfolio/data/gis-portal.png" style="display: block; margin: 0 auto 1em;"></img>
											<h5><b>Implementation in the GIS Portal</b></h5>
											&emsp;&ensp;A map representation of these measurements is needed for an overall look over the network status. In order to achieve this, an integration with the Gis Portal is needed. With the overall coverage level per floor, the map layer is coloured accordingly &#45; in resemblance with the &quot;Wifi_coverage&quot; layer in the &quot;IT_Equipment&quot; Gis Portal page.
